# Elixir implementation plan

**Read `plans/shared-context.md` first.** This document covers only Elixir-specific decisions.

---

## Goal

An Elixir Mix project in `elixir/` that compiles to a standalone `escript` executable. It reads `resume.yaml`, groups work entries by employer, and writes `docs/index.html`. Uses EEx (Embedded Elixir) for templating.

---

## Language version

- **Elixir 1.18** with **OTP 27** (latest stable as of mid-2025).
- Use the modern `mix new --script` or standard `mix new` approach. The project compiles to an escript — a self-contained binary runnable with `elixir resume_renderer`.

---

## Mix project setup

```sh
cd elixir
mix new resume_renderer --app resume_renderer
```

Configure escript in `mix.exs`:

```elixir
def project do
  [
    app: :resume_renderer,
    version: "0.1.0",
    elixir: "~> 1.18",
    deps: deps(),
    escript: [main_module: ResumeRenderer.CLI]
  ]
end
```

Build with `mix escript.build`, producing `./resume_renderer`.


**`yaml_elixir`** parses YAML into Elixir maps/lists/primitives. It is the standard choice in the ecosystem, wraps Erlang's `yamerl`, and handles the nested structures in the resume cleanly.

## Data model (`model.ex`)

Use Elixir `defstruct` for typed data. Write `from_map/1` functions on each module to construct structs from the raw YAML-parsed maps.

```elixir
defmodule ResumeRenderer.Model.WorkEntry do
  defstruct [
    :employer,
    :employer_group,   # maps from "employerGroup" key
    :position,
    :url,
    :start_date,       # maps from "startDate"
    :end_date,         # maps from "endDate"; nil = "Present"
    :summary,
    :location,
    highlights: [],
    keywords: []
  ]

  def from_map(m) when is_map(m) do
    %__MODULE__{
      employer:       m["employer"],
      employer_group: m["employerGroup"],
      position:       m["position"],
      url:            m["url"],
      start_date:     m["startDate"],
      end_date:       m["endDate"],
      summary:        m["summary"],
      location:       m["location"],
      highlights:     m["highlights"] || [],
      keywords:       m["keywords"] || []
    }
  end
end

defmodule ResumeRenderer.Model.Skills do
  defstruct sets: [], list: []

  def from_map(m) do
    %__MODULE__{
      sets: Enum.map(m["sets"] || [], &SkillSet.from_map/1),
      list: Enum.map(m["list"] || [], &SkillItem.from_map/1)
    }
  end
end

defmodule ResumeRenderer.Model.SkillSet do
  defstruct [:name, skills: []]
  def from_map(m), do: %__MODULE__{name: m["name"], skills: m["skills"] || []}
end

defmodule ResumeRenderer.Model.SkillItem do
  defstruct [:name, :level, :summary, :years]
  def from_map(m), do: %__MODULE__{
    name: m["name"], level: m["level"],
    summary: m["summary"], years: m["years"]
  }
end

defmodule ResumeRenderer.Model.Resume do
  defstruct [:basics, :disposition, work: [], projects: [], skills: nil,
             certificates: [], education: [], languages: [], interests: [],
             testimonials: [], references: []]

  def from_map(m) do
    %__MODULE__{
      basics:       Basics.from_map(m["basics"] || %{}),
      disposition:  Disposition.from_map(m["disposition"] || %{}),
      work:         Enum.map(m["work"] || [], &WorkEntry.from_map/1),
      projects:     Enum.map(m["projects"] || [], &Project.from_map/1),
      skills:       Skills.from_map(m["skills"] || %{}),
      certificates: Enum.map(m["certificates"] || [], &Certificate.from_map/1),
      education:    Enum.map(m["education"] || [], &Education.from_map/1),
      languages:    Enum.map(m["languages"] || [], &Language.from_map/1),
      interests:    Enum.map(m["interests"] || [], &Interest.from_map/1),
      testimonials: Enum.map(m["testimonials"] || [], &Testimonial.from_map/1),
      references:   Enum.map(m["references"] || [], &Reference.from_map/1)
    }
  end
end
```

Define all remaining structs (`Basics`, `Location`, `Profile`, `Disposition`, `Relocation`, `Project`, `Certificate`, `Education`, `Language`, `Interest`, `Testimonial`, `Reference`) following the same pattern.

## Employer grouping (`grouping.ex`)

```elixir
defmodule ResumeRenderer.Grouping do
  defstruct [:display_name, :former_names, :url, :start_date, :end_date, :positions]

  alias ResumeRenderer.Model.WorkEntry

  def group_work(entries) do
    entries
    |> Enum.reduce([], fn entry, acc ->
      key = entry.employer_group || entry.employer
      case acc do
        [current | rest] when group_key(current) == key ->
          [extend_group(current, entry) | rest]
        _ ->
          [new_group(entry) | acc]
      end
    end)
    |> Enum.reverse()
  end

  defp group_key(%__MODULE__{positions: [first | _]}) do
    first.employer_group || first.employer
  end

  defp new_group(%WorkEntry{} = e) do
    %__MODULE__{
      display_name: e.employer,
      former_names: [],
      url:          e.url,
      start_date:   e.start_date,
      end_date:     e.end_date,
      positions:    [e]
    }
  end

  defp extend_group(%__MODULE__{} = g, %WorkEntry{} = e) do
    former =
      if e.employer != g.display_name and e.employer not in g.former_names,
        do: g.former_names ++ [e.employer],
        else: g.former_names

    %{g |
      start_date:   min_date(g.start_date, e.start_date),
      former_names: former,
      positions:    g.positions ++ [e]
    }
  end

  # Lexicographic comparison works for ISO 8601 dates (YYYY-MM-DD)
  defp min_date(a, b) when a <= b, do: a
  defp min_date(_, b), do: b
end
```

## Date formatting (`date_format.ex`)

```elixir
defmodule ResumeRenderer.DateFormat do
  @months ~w[Jan Feb Mar Apr May Jun Jul Aug Sep Oct Nov Dec]

  def format(nil), do: "Present"
  def format(""),  do: "Present"
  def format(iso) do
    case String.split(iso, "-") do
      [year, month | _] ->
        idx = String.to_integer(month) - 1
        "#{Enum.at(@months, idx)} #{year}"
      [year] ->
        year
    end
  end
end
```

## Non-breaking space helper (`render.ex`)

```elixir
defmodule ResumeRenderer.Render do
  def nbsp_words(text) when is_binary(text) do
    words = String.split(text, " ")
    words
    |> Enum.with_index()
    |> Enum.map_join("", fn
      {word, i} when i < length(words) - 1 ->
        next = Enum.at(words, i + 1)
        sep = if String.length(word) <= 4 and String.length(next) > 4, do: "&nbsp;", else: " "
        word <> sep
      {word, _} ->
        word
    end)
  end
end
```

EEx does not auto-escape HTML in the same way as Jinja2. The template author is responsible for escaping. Use the `:html_escape` option in EEx or wrap values with `Plug.HTML.html_escape/1`. Since there is no HTTP framework here, implement a small escape function:

```elixir
defp h(nil), do: ""
defp h(s) when is_binary(s) do
  s
  |> String.replace("&", "&amp;")
  |> String.replace("<", "&lt;")
  |> String.replace(">", "&gt;")
  |> String.replace("\"", "&quot;")
end
defp h(v), do: h(to_string(v))
```

Call `h(value)` in the template for all user-supplied string values. Pre-escaped content (from `nbsp_words`, `format_date`) is inserted directly without `h()`.

## EEx template (`priv/template.html.eex`)

EEx uses `<%= expr %>` for output and `<% expr %>` for logic. It is pure Elixir — no DSL.

```eex
<!DOCTYPE html>
<html lang="en">
<head>
<!-- ... full <head> with CSS from docs/index.html ... -->
</head>
<body>
<div class="page">

  <header>
    <div>
      <h1 class="name"><%= h(basics.name) %></h1>
    </div>
    <div class="contact">
      <a href="mailto:<%= h(basics.email) %>"><%= h(basics.email) %></a><br>
      ...
    </div>
  </header>

  <section class="summary">
    <div class="section-label">Profile</div>
    <p><%= nbsp_words(h(basics.summary)) %></p>
  </section>

  <section>
    <div class="section-label">Experience</div>
    <%= for {group, gi} <- Enum.with_index(employer_groups) do %>
      <%= if length(group.positions) > 1 do %>
        <div class="employer-group">
          <div class="employer-header">
            <div>
              <span class="employer-name">
                <%= if group.url do %><a href="<%= h(group.url) %>"><% end %>
                <%= h(group.display_name) %>
                <%= if group.url do %></a><% end %>
              </span>
              <%= if group.former_names != [] do %>
                <span class="employer-former">
                  (formerly <%= h(Enum.join(group.former_names, ", ")) %>)
                </span>
              <% end %>
            </div>
            <span class="job-dates">
              <%= DateFormat.format(group.start_date) %> – <%= DateFormat.format(group.end_date) %>
            </span>
          </div>
          <%= for {pos, pi} <- Enum.with_index(group.positions) do %>
            <%= render_position.(pos, false) %>
            <%= if pi < length(group.positions) - 1 do %>
              <div class="position-divider"></div>
            <% end %>
          <% end %>
        </div>
      <% else %>
        <%= render_position.(hd(group.positions), true, group) %>
      <% end %>
      <%= if gi < length(employer_groups) - 1 do %>
        <hr class="job-divider">
      <% end %>
    <% end %>
  </section>

  <!-- ... projects, skills, education, testimonials ... -->

</div>
</body>
</html>
```

For the position partial, since EEx has no include mechanism, define a local function in `render.ex` that evaluates a sub-template string, or use a `render_position` anonymous function bound before calling `EEx.eval_file`. The cleanest approach for EEx is to define named functions and call them with `<%= ResumeRenderer.Render.position_html(pos) %>` — those functions build HTML strings using EEx evaluated at compile time via `EEx.function_from_file`.

```elixir
# In render.ex — compile a partial at compile time
require EEx
EEx.function_from_file(:def, :position_html, "priv/position.html.eex", [:pos, :show_employer, :group])
EEx.function_from_file(:def, :render_resume, "priv/template.html.eex",
  [:basics, :employer_groups, :projects, :skill_sets, :skill_map,
   :certificates, :education, :languages, :interests, :testimonials])
```

This compiles the EEx templates at compile time, making the binary efficient and catching template syntax errors during `mix compile`.

## `cli.ex` (escript entry point)

```elixir
defmodule ResumeRenderer.CLI do
  alias ResumeRenderer.{Model, Grouping, Render}

  def main(args) do
    {opts, _, _} = OptionParser.parse(args,
      strict: [input: :string, output: :string],
      aliases: [i: :input, o: :output]
    )
    input  = opts[:input]  || "../resume.yaml"
    output = opts[:output] || "../docs/index.html"

    {:ok, raw} = YamlElixir.read_from_file(input)
    resume = Model.Resume.from_map(raw)
    groups = Grouping.group_work(resume.work)
    skill_map = Map.new(resume.skills.list, fn item -> {item.name, item} end)

    html = Render.render_resume(
      resume.basics,
      groups,
      resume.projects,
      resume.skills.sets,
      skill_map,
      resume.certificates,
      resume.education,
      resume.languages,
      resume.interests,
      resume.testimonials
    )

    File.write!(output, html)
    IO.puts(:stderr, "wrote #{output}")
  end
end
```

## Notes

- `YamlElixir.read_from_file/1` returns `{:ok, map}` where map keys are strings (not atoms). Always access with string keys (`m["employer"]`), not atom keys.
- EEx compiled via `EEx.function_from_file` generates a module-level function — this is the idiomatic way to use EEx for production rendering. Do not use `EEx.eval_file` at runtime; it parses the template on every invocation.
- Since EEx does not HTML-escape by default, the `h/1` helper must be called on every user-supplied value in the template. Pre-formatted values (dates, `nbsp_words` output) do not need `h/1` because they are constructed from trusted string operations.
- Lexicographic comparison of ISO 8601 date strings is valid for sorting/comparison as long as all dates use the same format precision. The YAML consistently uses `"YYYY-MM-DD"`, so `min_date/2` using `<=` is correct.
- The escript includes the Elixir runtime but not OTP applications by default. `yaml_elixir` requires its application to be started; add `Application.ensure_all_started(:yaml_elixir)` at the top of `main/1`, or configure it in `mix.exs` escript options: `escript: [main_module: ..., app: nil]` and start manually.
- For the skills section, pass `skill_map` (a `Map` with skill name as key) to the template function. In EEx: `<%= skill_map[skill_name].level %>`.
